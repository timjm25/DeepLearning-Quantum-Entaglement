Quantum Entanglement — Theory

Background theory and key references for researchers using this tool.

What is Quantum Entanglement?

Quantum entanglement is a phenomenon where two or more particles become correlated such that the quantum state of each particle cannot be described independently of the others — even when separated by arbitrarily large distances. First described by Einstein, Podolsky and Rosen in 1935 as the “EPR paradox,” entanglement has since become the foundational resource for quantum computing, quantum cryptography, and quantum teleportation.

A bipartite quantum state ρ is entangled if it cannot be written as a product state ρ = ρA ⊗ ρB. For pure states, this is equivalent to the reduced density matrix being mixed (non-pure). For mixed states, determining separability is NP-hard in general — motivating the machine learning approach implemented here.

Entanglement Measures

Concurrence (Wootters, 1998)

The Wootters concurrence C(ρ) ∈ [0,1] quantifies entanglement for 2-qubit mixed states. C=0 means separable; C=1 means maximally entangled.

C(ρ) = max(0, λ₁ - λ₂ - λ₃ - λ₄)

where λᵢ are the eigenvalues in decreasing order of:
  R = √(√ρ · ρ̃ · √ρ)

and ρ̃ = (σy ⊗ σy) ρ* (σy ⊗ σy)   [spin-flip operation]

Von Neumann Entropy

S(ρ) = -Tr(ρ log₂ ρ) = -Σᵢ λᵢ log₂ λᵢ

For pure bipartite states: E(|ψ〉) = S(ρ_A) where ρ_A = Tr_B(|ψ〉〈ψ|)

Negativity (Peres-Horodecki)

N(ρ) = ( ∥ρ⁻Γₐ∥₁ - 1 ) / 2

ρ⁻Γₐ is the partial transpose of ρ with respect to subsystem A.
If N > 0: the state is entangled (NPT state).
For 2×2 and 2×3 systems, PPT ⇔ separable (Peres 1996, Horodecki 1996).

Deep Learning Architecture — EntanglementNet

A multi-task neural network trained on synthetically generated quantum states with analytically computed entanglement labels. The separability problem for mixed states is NP-hard in general; the network provides fast, approximate classification and continuous measure estimation.

Input: density matrix ρ (4×4 complex → 32 real features)
       [Re(ρ₀₀),...,Re(ρ₃₃), Im(ρ₀₀),...,Im(ρ₃₃)]

Backbone:
  Linear(32, 256) → BatchNorm → ReLU → Dropout(0.3)
  Linear(256, 128) → BatchNorm → ReLU → Dropout(0.2)
  Linear(128, 64)  → ReLU

Classification head:               Regression head:
  Linear(64, 32) → ReLU             Linear(64, 32) → ReLU
  Linear(32, 1)  → Sigmoid           Linear(32, 3)  → Sigmoid
  P(entangled) ∈ [0,1]             [concurrence, entropy, negativity] ∈ [0,1]

Multi-task loss:
  L = BCE(ŷ_cls, y_cls) + 0.5 × MSE(ŷ_reg, y_reg)

Training dataset: 8,000 states — Bell states, Haar-random pure states, product states, Werner states (all p values), and random mixed states. Adam optimiser with cosine annealing LR schedule over 40 epochs.
